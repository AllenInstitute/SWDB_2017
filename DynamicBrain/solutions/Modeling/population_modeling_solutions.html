Modeling a coupled excitatory-inhibitory (E-I) network with DiPDE/popnet

In this tutorial, we will demonstrate simulation of an E-I network consisting of homogenous excitatory (E) and inhibitory (I) populations using the DiPDE simulator.

In this example, only the excitatory (E) population is being driven by excitatory input from the external (Ext) population.

<img src="schematics_png/DiPDE_ei_net.png" alt = Drawing style = "width: 500px;">

Building the network

We will build the network step-by-step as follows:

1) Create/parameterize the nodes (populations) in the internal network, namely the E and I populations

2) Create/parameterize the external population.

3) Create/parameterize the edge (connection) from the external population to the E population.

4) Connect the E population to the I population

5) Run simulation and observe output

6) Exercise: Connect I to E and explore how the outputs change

What do these parameters correspond to?

approx_order: None,

dv: 0.0001 -> grid-size for the membrane potential (in volts)

record: True

tau_m: 0.0429 -> membrane time-constant for each neuron in the population (in seconds)

update_method': gmres -> internal solver-specific method

v_max: 0.02, -> threshold membrane potential (in volts)

v_min: -0.05 -> resting membrane potential (in volts)

Ext_input = NxNetwork("Ext_input")

Ext_input.add_nodes(node_type_id ='filter_001', ei='e', pop_name='input_filter', level_of_detail='filter')

input_directory_name = 'network/source_input/'

Ext_input.save_types(filename=input_models_file, columns=['node_type_id','level_of_detail','ei', 'pop_name'])

The steps used above are the same as those used for creating the E and I nodes.

The only difference is that we have introduced two additional parameters 'ei' and 'level_of_detail in creating this node.

Setting ei = 'e' specifies that the input is excitatory.

Setting level_of_detail = 'filter' specifies that the external input arises from filter level models.

This will not be important right now, but will become clear when we work through the biophysical example.

In [11]:
Ext_input.connect(source = Ext_input.nodes(pop_name = 'input_filter'),
            target = net.nodes(pop_name = 'excitatory'),
            edge_params={'edge_type_id':0,
                         'weight': 0.0025, 
                         'delay': 0.002, 
                         'nsyns': 10,
                         'params_file': 'InpToExc.json'} 
                         )

Let us unpack this a bit. To establish a connection, we provide the source (node named 'ext_excitatory' in Ext_input) and the target (node named 'excitatory' in net). Note that the pop_names for source and target should match the ones we specified when creating the nodes.

We also provide the parameters for the edge contained in edge_params. For DiPDE/popnet, each edge is specified by the following:

1) 'edge_type_id': label assigned to this edge; as long as this is unchanged, you can change any edge parameters without creating a new edge.

2) 'weight' : synaptic weight in volts (w in the cheat-sheet)

3) 'delay' : synaptic delay in seconds

4) 'nsyns' : number of synapses from source onto target.

5) 'params_file': optional filename to save parameters to.

5) Run simulation

Config file

At this point, we have created and saved a bunch of files (nodes, edges etc.). All of this needs to be integrated together, which is what is done in a config file. A default config file ('config.json') has already been created for us that contains:

1) runtime settings for the simulation in 'run'. We will see below how to change the default settings.

2) filenames for the different nodes and edges (we used these names keeping the directory structure specified here)

3) filenames for where the output should be stored.

Feel free to also look at the config file directly as it will look better than the print function below.
